Keepalived实现mongoDB的高可用

实现目标：

A(192.168.1.34)、B(192.168.1.33)两机器同时工作，互为主从，其中一台宕机，业务自动切换到另一台，不影响用户访问。

此篇是以mongoDB数据库的双向复制完成数据库的高可用性。

安装mongoDB

1、解压

2、添加path路径 vim /etc/profile

执行source /etc/profile，使profile生效。

3、创建配置文件：vim /etc/mongodb.conf

4、创建服务脚本vim /etc/init.d/mongodb

5、赋予脚本执行权限

6、注册为服务

MONGODB的双向复制

1、修改主服务A的配置文件:vim /etc/mongodb.conf

2、修改从服务B的配置文件: vim /etc/mongodb.conf

3、测试双向复制

启动两机器mongoDB服务后登录进去

可以看到两机器都为主服务。

    在B机器上查询

  • 3、把A服务停掉：service mongodb stop

    在B机器上添加一条数据

    再启动A服务：service mongodb start

    然后在A机器上查询

    可以看到数据已经从B服务上同步过来。

    已经实现预期目标。
